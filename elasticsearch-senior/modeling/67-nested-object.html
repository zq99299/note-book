# 基于 nested object 实现博客与评论嵌套关系

会使用 object 类型来实现一个需求：搜索 28 岁的黄药师的评论，然后引出为什么需要使用 nested objec 类型

# nested objec 类型

这次我们改用 nested objec 类型


DELETE /website

PUT /website
{
  "mappings": {
    "blogs": {
      "properties": {
        "comments": {
          "type": "nested",
          "properties": {
            "name":    { "type": "string"  },
            "comment": { "type": "string"  },
            "age":     { "type": "short"   },
            "stars":   { "type": "short"   },
            "date":    { "type": "date"    }
          }
        }
      }
    }
  }
}
